7. 限制与风险

#限制项严重程度说明
1APFS 已删除恢复为扫描级中等仅扫描 NXSB/APSB/B-tree 结构，未实现完整文件提取
2memory.connections 仅限 Linux中等基于 /proc/net/tcp，macOS/Windows 不可用
3Windows 设备枚举需 pywin32低core/platform.py 已有适配代码，需要安装 pywin32
4carving 为粗粒度信息基于签名扫描，不执行碎片重组或智能校验
5无 E01/AFF 映像格式信息需安装 libewf-python 或 pyaff
6hunt 非实时扫描信息一次读取整个文件到内存

8. 审计结论

总体评价: 通过

Forensic Toolkit v0.2.0 实现了一个完整的跨平台数字取证工具集。

核心亮点：

  • 零外部依赖 — 全部功能使用 Python 标准库实现，安装部署零摩擦
  • 11 个取证模块 — 覆盖文件系统、字符串、时间线、敏感信息、哈希、雕刻、元数据、网络、恢复、注册表、内存
  • 模块化架构 — ModuleBase + ModuleRegistry 使得新增模块只需实现一个类即可自动接入 CLI
  • 证据链 — 内置 Chain of Custody 记录，满足数字取证的可追溯性要求
  • 跨平台 — 三平台统一 CLI，PAL 隔离 OS 差异
